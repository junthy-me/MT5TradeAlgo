## Context

当前 `mt5/P4PatternStrategy.mq5` 的日志以全生命周期排查为目标，包含入场阻止、观察窗口、共享骨架、弱止损激活、平仓等多类打印。对调试细节有帮助，但对日常看盘和回放买点来说噪声过高。

同时，策略虽然在 `ENTRY` 日志里已经记录了 `P0-P4` 点位时间和价格，但这些信息埋在长字符串里，肉眼很难直接还原出“这笔单对应的是哪组模式”。用户想优先看到 P4 买点，并能在图上直接确认买点所使用的 `Pre0-P0-P1-P2-P3-P4`，以及后续若触发 `P5/P6` 时使用的是哪组延展点位，同时一眼分辨各点、关键高度值和止损位。

实现上有两个约束：
- 当前 EA 是多品种轮询模型，不一定只处理挂载图表本身的品种。
- MT5 图形对象只能画到具体 chart 上，因此需要定义“画到哪张图”和“没有匹配图时怎么办”。

## Goals / Non-Goals

**Goals:**
- 默认日志只突出成功买入的 `P4` 模式信息，减少非关键噪声输出。
- 为每次成功买入输出一条结构化、易读的买点摘要，明确标记 `P0-P4` 时间和价格。
- 在图表上绘制每次成功买入对应的 `Pre0-P4` 模式，并在首次激活时补充 `P5/P6`，让操作者可以直接看到这笔单使用的是哪组点位。
- 让 `Pre0/P0/P1/P2/P3/P4/P5/P6` 各点使用固定且彼此不同的颜色，便于跨多次买入快速辨认。
- 在图上补充 `Pre0-P0` 下跌值、`b1`、`a`、`b2`、`c` 的数值标注，以及强止损/弱止损水平位。
- 让多次买入的图形标注彼此可区分，不会因对象重名互相覆盖。

**Non-Goals:**
- 不改变模式识别、入场门控、止损止盈或持仓管理逻辑。
- 不在本次变更中重新设计完整的调试日志体系。
- 不自动为未打开的品种/周期创建新图表。
- 不在本次变更中绘制完整的持仓生命周期轨迹或出场点。

## Decisions

### 1. 默认只保留成功 P4 买点摘要日志

`ENTRY` 日志保留，并改成更短、更聚焦的格式，至少包含：
- `symbol`
- `ticket`
- `p4_bar`
- 成交价
- `P0-P4` 各点的时间和价格
- 必要的风控价位，如 `hard_loss`

默认移除或静默以下“常规运行日志”：
- 入场被观察窗口或共享骨架阻止的日志
- 弱止损/止盈首次激活日志
- 常规 `EXIT` 生命周期详情日志

理由：用户当前目标是先看清楚买点匹配了什么模式，而不是做生命周期级别排障。保留单条买点摘要可以大幅降低 Experts 输出密度。

备选方案：
- 保留全部日志但仅缩短字段。这个方案仍然无法解决日志总量过大问题。
- 增加复杂的多级日志配置。这个方向合理，但超出这次“先收缩默认输出”的目标。

### 2. 图表标注只画到已打开且匹配的 chart

每次成功创建受管仓位后，策略尝试查找已打开的、`symbol == entry.symbol` 且 `period == InpTF` 的 chart，并在该 chart 上绘制模式对象。

如果没有找到匹配 chart：
- 本次不自动 `ChartOpen()`
- 仅保留轻量提示或静默跳过，避免策略在多品种模式下自动打开大量图表

理由：自动开图虽然更“显眼”，但对多品种扫描 EA 来说会造成明显的界面副作用。先要求用户打开关心的图，再在该图上看到模式，更可控。

备选方案：
- 自动打开目标 chart 并绘图。优点是更直接，缺点是会改变用户工作区布局。
- 只在挂载 chart 上绘图。对于多品种扫描场景不成立。

### 3. 每次买入使用独立对象组绘制 `Pre0-P6`

每次成功买入创建一组对象，命名空间包含：
- 固定前缀
- `symbol`
- `timeframe`
- `ticket`
- `p4_bar_time`

对象组至少包括：
- `Pre0-P4` 六个点位标记
- 首次激活后补充的 `P5/P6` 点位标记
- 相邻点之间的连线
- `P4` 买点箭头或强调标记
- 关键高度值文字标注：`Pre0-P0` 下跌值、`b1`、`a`、`b2`、`c`
- 强止损和弱止损水平标注

理由：以 `ticket + p4_bar_time` 作为唯一键，可以保证同一品种多次买入的图形不会互相覆盖，同时能从图上反查到对应交易。

备选方案：
- 仅保留一套最新模式对象。这样会覆盖历史买点，不利于回放。
- 直接把所有信息放进一条 `OBJ_TEXT`。可读性较差，也不够直观。

### 4. 图形标注覆盖入场和首次激活后的关键结构，并为每个点位使用固定颜色

本次变更把“买入时到底匹配了哪个模式”可视化到 `Pre0-P4`，并在持仓首次激活 `P5/P6` 后继续补充 `P5/P6` 与弱止损位。其中每个点位颜色固定，且不同点位之间颜色不同，例如同一个 `P1` 在不同交易里始终使用同一种颜色。

理由：用户当前的核心问题是确认每次买入所用的模式，同时快速分辨每个点位和关键高度。固定配色比按“高点/低点”共用颜色更容易跨多次买入比较。

## Risks / Trade-offs

- [多品种扫描但未打开对应图表] → 不会看到图上标注；通过在 README 中明确“需要先打开对应品种/周期图表”来降低误解。
- [默认日志减少后，排障信息不足] → 本次先满足看盘需求；后续如需要，可再起 change 增加可切换的 debug 日志模式。
- [图形对象数量持续累积] → 使用稳定命名空间，后续可再补清理策略；本次先保证历史买点不会被覆盖。
- [对象绘制失败但买单已成功] → 绘图失败不能影响交易本身，绘图逻辑必须与交易管理解耦。

## Migration Plan

1. 收缩默认日志输出，仅保留成功 `P4` 买点摘要。
2. 在成功买入后增加 chart annotation 流程。
3. 更新 README，说明新的日志格式和图表使用方式。
4. 在 MT5 中验证单图、多次买入和未打开匹配图表时的行为。
