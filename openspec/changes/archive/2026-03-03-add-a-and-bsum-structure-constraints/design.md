## Context

当前完整匹配在历史骨架阶段主要依赖 `CondA` 和单段跨度上限，在实时 `P4` 阶段主要依赖 `CondB` 与 `CondC`。这意味着只要 `b1/b2` 比例合适，且 `P3-P4` 回落够深，某些 `a` 极小、`P1-P2` 持续时间极短，或者 `b1+b2` 相对 `a` 不够展开的骨架仍会进入后续流程。

用户希望恢复一部分更细的“结构质量”筛选，但这些筛选应明确限定在历史骨架阶段，而不是重新回到旧的 `c < m*a` 或入场噪声过滤逻辑。

## Goals / Non-Goals

**Goals:**
- 为 `P1-P2` 线段增加最小价格幅度约束
- 为 `P1-P2` 线段增加最小持续 K 线数量约束
- 为 `b1+b2` 相对 `a` 增加最小比例约束
- 让这些约束在参数名、spec 和日志语义上都可直接解释

**Non-Goals:**
- 不恢复旧的 `CondD` 公式
- 不恢复 `b1+b2 / buyPrice` 噪声过滤
- 不改变 `P4` 实时触发、止盈或弱止损的既有公式

## Decisions

### 1. 三条新约束全部落在历史骨架阶段

实现位置放在 `BuildHistoricalBackbone()`，因为这里已经拿到了：

- `a = p1 - p2`
- `b1 = p2 - p0`
- `b2 = p3 - p1`
- `pointSpans[1] = i2 - i1`

这样可以在候选进入缓存前就完成过滤，避免把本就不合格的骨架继续带入实时 `P4` 评估。

### 2. `P1-P2` 时间约束按“包含端点的 K 线数”解释

当前 `pointSpans[1] = i2 - i1` 表示两点之间的索引差。用户希望的口径是“包含 `P1` 和 `P2` 本身的 K 线数量”，因此实现等价为：

```text
pointSpans[1] + 1 >= InpP1P2AValueTimeMinKNumberLimit
```

默认值 `3` 时，实际要求 `i2 - i1 >= 2`。

### 3. `a` 的最小空间约束使用价格单位

`InpP1P2AValueSpaceMinPriceLimit` 直接与 `a` 的价格值比较，不转换为 points。这样和现有 `a/b1/b2/c` 的计算口径一致，也避免在不同 symbol 上混入额外换算层。

### 4. `bSum` 约束使用相对 `a` 的最小比例

采用：

```text
(b1 + b2) >= InpBSumValueMinRatioOfAValue * a
```

而不是单独给 `b1+b2` 设一个绝对最小价格值。这样能更稳定地表达“左右两侧结构需要相对 `a` 足够展开”的意图。

### 5. `a` 的语义文档明确为“有限回归”

当前主 spec 强调 `a` 不再作为独立门槛。此次变更需要明确修正：`a` 不再参与旧式即时风控或止盈/止损推导，但可以在历史骨架阶段用于最小空间、最小时间和相对比例筛选。

## Risks / Trade-offs

- [新增三条约束会显著减少候选骨架数量] -> Mitigation: 保留日志中的 `a`、`b1`、`b2` 和 `P1-P2` span，便于回测对照。
- [价格单位阈值在不同 symbol 上尺度差异明显] -> Mitigation: 先按用户要求保留价格单位；若后续跨品种使用困难，再单独提 change 讨论 points 化。
- [这次变更与前一轮“弱化 a 的独立门槛”方向部分相反] -> Mitigation: 在 `mt5-pattern-a-simplification` 里明确记录这是限定在历史骨架阶段的受控回归。
