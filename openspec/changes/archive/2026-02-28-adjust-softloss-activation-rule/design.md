## Context

弱止损目前在入场后形成 `P5` 和 `P6` 时激活，止损价位本身仍保持为 `soft_loss_price = softLossC * Price_P5`。本次变化不涉及弱止损价位公式，也不调整 `P5/P6` 的识别方式，只改变“什么时候允许激活弱止损”的判断逻辑，并同步默认参数值。

当前新的目标规则是：

```text
旧条件: (e - d) >= InpSoftLossN * c
新条件: e >= InpSoftLossN * (c + d)
```

其中：
- `c = P3P4` 高度
- `d = P4P5` 高度
- `e = P5P6` 高度

## Goals / Non-Goals

**Goals:**
- 将弱止损激活条件统一为 `e >= InpSoftLossN * (c + d)`。
- 将 `InpSoftLossN` 的默认值统一为 `0.65`。
- 保持弱止损价位公式、`P5/P6` 识别方式和弱止损触发平仓逻辑不变。

**Non-Goals:**
- 不修改 `soft_loss_price = softLossC * Price_P5`。
- 不修改强止损、止盈或入场规则。
- 不修改 `P5/P6` 的角色化点位来源、搜索顺序或跨度语义。

## Decisions

### 1. 弱止损激活改为“总回收比例”判断

弱止损激活条件统一为：

```text
e >= InpSoftLossN * (c + d)
```

原因：
- 这个表达直接衡量 `P5P6` 的反弹高度是否覆盖 `P3->P4->P5` 总跌幅的一定比例，语义更直观。
- 它避免了旧条件 `(e - d)` 这种“净超出量”表达带来的理解成本。

备选方案：
- 保留旧公式 `(e - d) >= n * c`。否决原因是已被用户明确替换。

### 2. 默认 `InpSoftLossN` 提高到 0.65

运行时默认参数将改为 `InpSoftLossN = 0.65`。

原因：
- 这是当前规则调整的一部分，默认配置需要和新判断公式一起生效。

备选方案：
- 保持默认值 `0.5` 不变，仅替换公式。否决原因是与用户指定的目标默认值不一致。

## Risks / Trade-offs

- [新公式与旧公式的激活区间不同，历史回测结果会变化] -> Mitigation: 在验证任务中显式检查弱止损激活时机，并与旧规则预期区分。
- [只修改实现不更新 spec 会造成再次漂移] -> Mitigation: 将此变化单独作为 spec change 记录并后续归档。

## Migration Plan

1. 更新交易管理 spec，明确弱止损激活条件改为 `e >= n * (c + d)`。
2. 更新实现中的 `InpSoftLossN` 默认值和激活判断。
3. 编译并通过回测验证新条件按预期激活弱止损。

Rollback:
- 恢复到 `(e - d) >= n * c` 和默认值 `0.5`。

## Open Questions

- 暂无。弱止损价位公式本次保持不变。
